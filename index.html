BSO ASNR — Mise en place et retour d’expérience

Rapport de stage (M1 GSI, Paris 8) : construction d’un Baromètre de la Science Ouverte pour l’ASNR, comparaisons Scopus/OpenAlex/HAL, scripts Python et recommandations opérationnelles.

Résumé exécutif

Objectif : établir un BSO institutionnel pour l’ASNR (ex‑IRSN) sur 2013–2024 en consolidant des extractions Scopus, OpenAlex et HAL, puis en déposant/complétant les manquants dans HAL pour fiabiliser le taux d’Open Access.

Contexte

Fusion ASN+IRSN → ASNR (2025). Au SEARCH/USNR : besoin d’un suivi fiable de la production scientifique et de l’OA. Le BSO national est utile mais partiellement opaque pour un usage interne fin.

Approche

Base maître (7092 lignes V1), croisements API (OpenAlex, HAL, Unpaywall), scripts Python ad hoc, vérifications manuelles, dépôts HAL ciblés (X2HAL + dépôts directs).

Impact

OpenAlex couvre >70% des publications et révèle ~20% hors Scopus. Processus reproductible, montée en compétences Python, feuille de route OA/BSO réaliste.

Problématique et hypothèses

Problématique

Comment fiabiliser un BSO institutionnel pour l’ASNR à partir de sources hétérogènes (Scopus, OpenAlex, HAL) en limitant le bruit (faux positifs d’affiliation) et en maximisant le dépôt OA réel ?

Hypothèses
  • H1 : Un pipeline « DOI‑first » (DOI, type_crossref, OA status) + curation réduit fortement le bruit vs requêtes par affiliation texte.
  • H2 : OpenAlex, combiné à HAL et Unpaywall, permet d’atteindre une couverture ≥ Scopus pour le périmètre ASNR, à condition de corriger typologies et affiliations.

Méthodologie

Design

Constitution d’une base maître (2013–2024). Dédoublonnage par DOI, contrôle typologies (préférer type_crossref), priorisation articles/chapitres/communications.

Collecte

APIs OpenAlex/Scopus/HAL, ajout OA/licence via Unpaywall, normalisation des champs pour X2HAL.

Analyse

Croisements, vérifs manuelles, dépôts HAL, reporting intermédiaire (2013, 2023, 2024) en vue BSO.

Résultats clés

Couverture & apports

  • Base V1 : 7092 lignes (2013–2024). DOI uniques : 1713 ; doublons : 4620 ; triplons : 9 ; sans DOI : 750.
  • OpenAlex couvre >70% et remonte ~20% non présentes dans Scopus.
  • Scripts : enrichissement HAL id / OA / APC, matching DOI→HAL, extraction OpenAlex par lot.

Points de vigilance

  • BSO : indicateurs utiles mais peu transparents → ne pas l’utiliser comme seule référence interne.
  • Affiliations : requêtes texte = bruit → préférer DOI + contrôles (idHAL/ORCID/ROR).
  • Typologies : éviter attribut type OpenAlex, privilégier type_crossref.
